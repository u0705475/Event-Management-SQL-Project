# Event-Management-SQL-Project
Group 2 Database Design Project: Conceptual and relational modeling for an event management system. Includes entity‑relationship diagrams, weak‑entity analysis, and schema documentation developed for IS 6420 at the University of Utah. Demonstrates SQL‑based data modeling, normalization, and collaborative design principles.

This repository contains the Group 2 presentation materials for the IS 6420 Database Theory and Design course.
The project models an event‑management database, covering conceptual design, entity relationships, and schema implementation.
Key deliverables include:

ER diagrams illustrating strong and weak entities (Event, Ticket, Attendee, etc.)

Normalized schema documentation and SQL table definitions

Presentation slides summarizing design decisions and check‑in milestones
